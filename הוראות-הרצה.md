# הוראות הרצה ידנית - מערכת Ein Tsofia

## דרישות מקדימות
- Node.js (גרסה 20 ומעלה)
- npm (מותקן יחד עם Node.js)
- חיבור לאינטרנט

## שלבי הרצה

### 1. התקנת תלויות (פעם ראשונה בלבד)
פתח טרמינל בתיקיית הפרויקט והרץ:
```bash
npm install
```

### 2. הגדרת משתני סביבה
וודא שקיים קובץ `.env.local` בתיקיית הפרויקט עם ההגדרות הבאות:
- הגדרות Firebase (אימות ואחסון)
- מפתח API של Gemini
- הגדרות נוספות

**הקובץ `.env.local` כבר קיים במערכת** ✓

### 3. הרצת שרת הפיתוח
הרץ את הפקודה הבאה:
```bash
npm run dev
```

### 4. גישה לאפליקציה
לאחר הרצת השרת, פתח דפדפן וגש לכתובת:
```
http://localhost:3000
```

## פקודות נוספות

### בניית גרסת ייצור
```bash
npm run build
```

### הרצת גרסת ייצור
```bash
npm run start
```

### בדיקת קוד (Linting)
```bash
npm run lint
```

## פתרון בעיות נפוצות

### השרת לא עולה
- ודא ש-Node.js מותקן: `node --version`
- וודא שהתלויות מותקנות: `npm install`
- נסה למחוק את תיקיית `.next` והרץ שוב: `rm -r .next` (או `rmdir /s .next` ב-Windows)

### שגיאות Firebase
- וודא שקובץ `.env.local` מכיל את כל המשתנים הנדרשים
- בדוק שמפתחות ה-API תקינים

### פורט 3000 תפוס
אם הפורט 3000 תפוס, Next.js יציע פורט חלופי אוטומטית (לדוגמה: 3001)

## מבנה הפרויקט
- `src/` - קוד המקור
  - `app/` - נתיבים ודפים (Next.js App Router)
  - `components/` - קומפוננטות React
  - `lib/` - ספריות עזר (Firebase, Gemini, וכו')
- `public/` - קבצים סטטיים
- `.env.local` - משתני סביבה (לא נכלל ב-Git)

## תמיכה
לשאלות או בעיות, פנה למפתח המערכת.
